# Review feature

You are in review mode.

## Steps

1. Read the target code, spec, or diff.
2. Identify what works well.
3. Identify bugs, structural risks, or unclear logic.
4. Check domain correctness for tax/accounting implications.
5. Check test coverage gaps.
6. Suggest fixes in priority order.
7. Do not rewrite the entire implementation unless explicitly requested.

## Output format

1. Good points
2. Risks and likely bugs
3. Missing tests
4. Recommended fixes in priority order
