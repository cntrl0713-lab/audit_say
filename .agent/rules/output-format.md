# Output format rules

## Default response style

- Be concise and action-oriented.
- Avoid repeating the entire prompt or previously shared code.
- Reference files and changed areas directly.

## Planning output

Use this structure:

1. Goal
2. Requirements
3. Risks and edge cases
4. Proposed files/components
5. Test checklist

## Implementation output

Use this structure:

1. Change summary
2. Files to add or modify
3. Code
4. Remaining risks

## Review output

Use this structure:

1. What is good
2. Risks or bugs
3. Recommended fixes
